Customer Portal — API Flow Diagrams

Complete request journey: Browser → Next.js Rewriter → API Gateway → Auth → Microservice

Master Architecture
All services connected through API Gateway with session verification on every authenticated request
API Gateway Middleware Pipeline
Every request passes through these steps before reaching a microservice
1
Incoming Request
From Next.js → /api/v1/...
↓
2
Inject Correlation ID
x-correlation-id (new or propagated)
↓
3
Strip Spoofed Headers
x-user-id, x-workspace-id, x-permissions, x-internal-service-secret
↓
4
Is this a public route?
/auth/login, /auth/signup, /auth/google/url, /auth/facebook, /auth/request-pwd-reset, /auth/reset-password
YES → skip to step 7 ↓ NO
5
Verify Session
POST auth-service:3006/internal/v1/auth/verify-session { token }
↓ 401 → reject | 200 → continue
6
Inject Enriched Headers
x-user-id · x-user-role · x-user-system-role · x-workspace-id · x-permissions · x-internal-service-secret
↓
7
Apply Path Rewrite Rule
e.g. /api/v1/automation → /api/automation | /api/v1/workspace → /provider/v1/workspace
↓
8
Proxy to Downstream Service
Forward enriched request → microservice port
