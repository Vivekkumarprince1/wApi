# Inbound Events Overview

Inbound events are triggered by the WhatsApp Self-Serve Platform to your callback URL.

## Classification of Events
1. **User Events**: Customer interactions like opting in/out or starting a sandbox session.
2. **System Events**: Platform-level updates like account activation (`partner-app-live`).
3. **Message Events**: Real-time status updates (sent, delivered, read) and incoming user messages.
4. **Billing Events**: Wallet deductions and threshold alerts.

[Source: Gupshup Documentation](https://partner-docs.gupshup.io/docs/inbound-events)
