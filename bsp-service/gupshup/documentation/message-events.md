# Message Events

Message events inform you about the real-time status of outgoing messages and incoming interactions.

## Key Payload Descriptions
- **whatsappMessageId**: The ID generated by Meta for the message.
- **gsId**: Unique Gupshup identifier for the message.
- **type**: Indicates the state (e.g., enqueued, sent, delivered, read, failed).

## Status Types
- **enqueued**: Message reached Gupshup/WhatsApp docker.
- **sent**: Message is in transit to the end-user. Includes a `pricing` object with `policy` (CBP/PMP) and `category`.
- **delivered**: Reached user's device.
- **read**: User viewed message (requires read receipts).
- **failed**: Includes `code` and `reason` (e.g., "Number Does Not Exist On WhatsApp").
- **mismatch**: Triggered when the destination number doesn't match the WhatsApp ID.

## Conversation Context
Messages include a `conversation` object with:
- **id**: Unique conversation ID.
- **expiresAt**: Expiration timestamp.
- **type**: Category (Marketing, Utility, etc.).
