# Passthrough APIs - Flow Management

Passthrough APIs allow partners to use Meta's raw format for sending messages, ensuring faster access to new features.

## Flow Management Lifecycle
1. **Create Flow**: Initialize a new flow structure.
2. **Update JSON**: Upload components and structured data.
3. **Get/Preview**: Retrieve flow details or a preview URL for testing.
4. **Publish**: Finalize the flow (cannot be edited after publishing).
5. **Deprecate/Delete**: Manage older or unused flows.

## Sending Flows
- Supports both **Individual (Session)** and **Template** message types through a dedicated V3 endpoint.
